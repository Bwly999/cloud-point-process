# 光谱共焦 PNG 点云图处理设计

## 1. 目标

针对光谱共焦设备输出的 `16-bit` 单通道 PNG 高度图，构建一条可重复执行的处理链路，完成以下目标：

- 消除蛇形拼接 3 道扫描之间的接缝高度偏移，避免接缝伪差污染梯度和曲率结果。
- 针对 `X/Y` 像素物理分辨率差异较大的问题，先对高采样方向做分步重采样，使两个方向的采样尺度接近一致。
- 计算并导出：
  - `X/Y` 向梯度图
  - `X/Y` 向二阶导图
  - `X/Y` 向剖面曲率图
- 同时输出可视化图片和 CSV 数值矩阵。
- 在没有真实输入图的情况下，先自动生成一张带拼接偏移的假高度图，用于验证算法效果。

## 2. 已知条件与物理定义

输入图像：

- 宽高为 `w * h`
- `16-bit` 单通道 PNG
- 高度换算关系：`z_mm = gray * 0.0001`

物理分辨率：

- `dx = 0.08 mm/pixel`
- `dy_raw = 0.005615 mm/pixel`
- `dz = 0.0001 mm/pixel`

扫描方式：

- 单次扫描宽度为 `11.5 mm`
- 对应 `2048` 像素
- 总共蛇形扫描 `3` 道
- 总高度 `h = 6144`，因此每道高度为 `2048`
- 已知扫描带接缝位于原图 `Y = 2048` 和 `Y = 4096`

## 3. 总体处理流程

处理流程按以下顺序执行：

1. 读取 `16-bit` 单通道 PNG，并转换为毫米高度矩阵 `Z(mm)`。
2. 针对 3 道扫描带的拼接偏移做整带补偿，并在接缝邻域做窄窗口平滑过渡。
3. 在接缝补偿后的高度图上进行轻量去噪平滑，以抑制微分运算对高频噪声的放大。
4. 沿 `Y` 方向按 `16:1` 做分步重采样，使 `dy` 接近 `dx`。
5. 在重采样后的高度图上按真实物理尺度计算一阶导、二阶导和剖面曲率。
6. 将处理过程和结果分别保存为 PNG 与 CSV。
7. 生成一张假高度图并跑通整条流程，用于验证接缝处理前后及导数结果变化。

## 4. 接缝补偿设计

### 4.1 问题建模

拼接伪差不被视为仅发生在接缝几行上的局部异常，而被建模为：

- 第 2 道扫描带相对第 1 道存在沿 `X` 变化的整体高度偏移
- 第 3 道扫描带相对第 2 道存在沿 `X` 变化的整体高度偏移

因此，补偿的主对象是“整条扫描带”，而不是只修接缝附近少量像素。

### 4.2 偏移估计

对每条接缝分别估计一条随 `X` 变化的低频偏移曲线 `Δ(x)`：

- 第一条接缝：估计 `Δ1(x)`，表示第 2 道相对第 1 道的偏移
- 第二条接缝：估计 `Δ2(x)`，表示第 3 道相对第 2 道的偏移

估计方法：

- 在接缝上方和下方分别取固定高度窗口，例如各 `64` 行
- 对每个 `X` 列，分别计算上方窗口和下方窗口的中位数高度
- 两者之差作为该列的原始偏移估计
- 再沿 `X` 方向对偏移曲线做稳健低频平滑，滤除局部真实形貌造成的高频扰动

选择中位数的原因：

- 对局部突刺、坑点、孤立毛刺更稳健
- 更不容易把单点缺陷误当成扫描带整体偏移

### 4.3 整带补偿

补偿方式采用累积式：

- 第 1 道：不补偿
- 第 2 道：减去 `Δ1(x)`
- 第 3 道：减去 `Δ1(x) + Δ2(x)`

这样可以将 3 道扫描带统一对齐到第 1 道的高度基准。

### 4.4 接缝软过渡

整带补偿完成后，在接缝邻域再增加窄窗口平滑过渡，例如前后各 `16` 到 `32` 行：

- 作用不是承担主补偿
- 作用是避免补偿后接缝处仍存在数值硬边
- 过渡采用连续权重函数，使上下两侧平滑连接

该步骤应严格限制在窄区域内，避免对真实形貌产生明显展宽。

## 5. 重采样设计

### 5.1 原因

原始采样尺度存在显著不均衡：

- `dx = 0.08`
- `dy_raw = 0.005615`

如果直接在该网格上计算 `X/Y` 导数，两个方向的数值尺度和噪声敏感性会明显不一致。

### 5.2 方法

沿 `Y` 方向执行 `16:1` 分步重采样：

- 新步长 `dy = 0.005615 * 16 = 0.08984 mm/pixel`

这样得到的 `dy` 与 `dx = 0.08` 接近，可认为近似等尺度采样。

### 5.3 实现倾向

优先采用窗口平均或抗混叠降采样，而不是简单抽点：

- 简单抽点会放大别名误差
- 窗口平均更适合高度场的物理量保持

## 6. 导数与曲率定义

所有导数与曲率均基于重采样后的高度矩阵计算，并使用真实物理间距 `dx` 与 `dy`。

### 6.1 梯度

输出两个方向的一阶导：

- `gx = dz/dx`
- `gy = dz/dy`

解释：

- 单位为 `mm/mm`
- 对应表面沿两个方向的坡度

### 6.2 方向二阶导

输出两个方向的二阶导：

- `gxx = d2z/dx2`
- `gyy = d2z/dy2`

解释：

- 单位为 `1/mm`
- 更偏工程中的“方向弯曲强度”
- 相比真正几何曲率更稳定

### 6.3 方向剖面曲率

将每一行或每一列视为二维剖面曲线，分别计算两个方向的剖面曲率：

- `kx = zxx / (1 + zx^2)^(3/2)`
- `ky = zyy / (1 + zy^2)^(3/2)`

解释：

- 单位为 `1/mm`
- 比二阶导更接近几何意义上的曲率
- 对噪声更敏感，因此必须在求导前做轻量平滑

### 6.4 本次不作为主输出的曲率

以下曲率不作为本次主结果：

- 高斯曲率 `K`
- 平均曲率 `H`

原因是当前需求明确要求 `X/Y` 方向分别出图，而 `K` 与 `H` 属于两个方向耦合后的曲面量，不符合主要交付目标。

## 7. 数值实现细节

### 7.1 导数离散形式

默认采用：

- 内部点使用中心差分
- 边界点使用一侧差分

这样实现简单、稳定，且便于解释单位和结果来源。

### 7.2 轻量平滑

在“接缝补偿后、求导前”增加弱平滑：

- 目标是抑制测量噪声和量化噪声
- 不允许明显改变宏观形貌

候选方式：

- 小窗口高斯平滑
- 小窗口均值平滑

默认优先使用小尺度高斯平滑。

### 7.3 参数外置

以下参数应允许配置：

- 每道扫描高度，默认 `2048`
- 接缝上下偏移估计窗口，默认 `64`
- 接缝过渡窗口，默认 `16` 或 `32`
- 偏移曲线沿 `X` 的平滑窗口
- `Y` 向降采样倍率，默认 `16`
- 导数前平滑强度

## 8. 假数据验证设计

在无真实输入图时，先生成一张合成高度图用于测试。

### 8.1 假图特征

假图应同时包含：

- 缓慢变化的背景曲面
- 局部凸起和凹坑
- 高频小幅纹理或噪声
- 3 道扫描带
- 第 2、3 道分别叠加随 `X` 缓慢变化的高度偏移

这样能够验证：

- 接缝补偿是否能压平拼接伪差
- 梯度与曲率是否主要反映真实形貌，而不是被接缝主导

### 8.2 验证关注点

应重点对比：

- 原始高度图与补偿后高度图
- 补偿前后接缝截面
- 梯度图是否仍出现明显整条接缝高亮
- 曲率图是否仍在接缝处形成大面积伪响应

## 9. 输出文件设计

程序输出至少包含以下文件：

- `height_raw_preview.png`
- `height_corrected_preview.png`
- `height_resampled_preview.png`
- `grad_x.png`
- `grad_x.csv`
- `grad_y.png`
- `grad_y.csv`
- `curv2_x.png`
- `curv2_x.csv`
- `curv2_y.png`
- `curv2_y.csv`
- `curve_x.png`
- `curve_x.csv`
- `curve_y.png`
- `curve_y.csv`

可选增强输出：

- 接缝截面前后对比图
- 偏移曲线 `Δ1(x)`、`Δ2(x)` 可视化
- 运行参数记录文件

## 10. 错误处理

程序应检查以下错误条件：

- 输入不是 `16-bit` 单通道 PNG
- 输入高度不足以容纳 3 道 `2048` 高度带
- 重采样倍率与图像高度不匹配
- 输出目录不存在但无法创建

当发生错误时，应明确指出：

- 哪一步失败
- 当前收到的图像尺寸与期望尺寸
- 哪个参数不合法

## 11. 测试策略

测试覆盖以下层级：

- 假数据生成测试
- 接缝补偿前后高度连续性测试
- `16:1` 重采样后尺寸与物理步长测试
- 梯度与二阶导维度测试
- CSV 与 PNG 输出完整性测试

重点回归风险：

- 接缝补偿误伤真实台阶
- 重采样顺序错误导致导数方向含义混乱
- 导数计算未使用真实物理间距
- 曲率结果受微小噪声过度放大

## 12. 当前限制与后续扩展

当前设计默认：

- 输入为固定 3 道扫描
- 接缝位置已知
- 高度编码为 `16-bit` 单通道

后续可扩展方向：

- 自动检测接缝位置
- 支持任意扫描带数量
- 支持直接处理 TIFF 或原始数值矩阵
- 增加高斯曲率和平均曲率作为附加输出
